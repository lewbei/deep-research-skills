# Contributing to deep-research-skills

Thank you for your interest in improving this project! We welcome contributions to refine this research and planning workflow.

## How to Contribute

### 1. Reporting Bugs & Feature Requests
- Check the issue tracker to ensure your topic hasn't already been discussed.
- Open a new issue with a clear description, reproduction steps (if applicable), and proposed solutions.

### 2. Submitting Pull Requests
1. Fork the repository and create your branch from `main`.
2. Ensure your changes align with the core philosophy of the interleaved loop, time budgets, and evidence validation.
3. Keep documentation and templates consistent. If you modify a template, make sure references in the `docs/` and `SKILL.md` are updated accordingly.
4. Commit your changes with clear, descriptive commit messages.
5. Push to your fork and submit a pull request.

## Style Guide for Skills & Workflows
- **Markdown Alignment:** Keep markdown documents well-organized with standard GitHub Flavored Markdown (GFM).
- **Subagent Boundaries:** Research subagents must remain strictly read-only.
- **Traceability:** Every claim must remain traceably connected to its original source.
